HTML and the Web

World Wide Web

The Web is composed primarily of a collection of HTML documents linked by hyperlinks. The web was invented in the early 1990's by Tim Berners-Lee and others. The Web is composed of computers that are optimized for running continuously to host and serve-up webpages (HTML documents). Called servers these computers receive requests sent to them by humans and their webbrowsers which send requests in the form of the protocol of the web, Hyper Text Tranfer Protocol or HTTP.

Hyper Text Markup Language

Hyper Text Markup Language (HTML) is the the main document type of the Web although other types of files can also be found--and linked to--files such as images, video, audio files, and document files besides HTML (for example, pdf documents). HTML is made up of:

1) the text content (what you see on the webpage through your browser), and
2) mark-up code that tells your browser what the text will look like or how it will be displayed.

HTTP Protocol

The internet uses HTTP to communicate with you, your browser, and servers. For example, when you enter the url of, say, Amazon.com into your browser, your browser sends a request in HTTP to the server that hosts Amazon's website.

Specifics of HTML

HTML is composed of plain text characters and tags. Tags are a kind of code that webbrowsers can understand and use to render or draw the various elements of the webpate. In HTML, plain text is rendered as plain text. HTML markup tags are enclosed in angled brackets like this:

<html-tag>

Some types of HTML tage require both an opening and a closing tag. HTML tags that do not require a closing tag are called void tags. The break tag <br> is an example of a void tag. It's easy to forget to add the closing tag for non-void tags, but you will often see the result of this error show up on the webpage, and a good text editor, such as scratchpad.io will automatically create the end tag as soon as you write the closing angled bracket on the opening tag.

Key Techniques and Methods

HTML Attributes - new concept

Tag attributes are written in the opening tag(in the first set of angled brackets) after the letter or letters specifying the type of tag. For example, an anchor tag that anchors a link has the opening tag thusly:
<a tag-attribute-here="some url webaddress here">. A tag attribute always equals some value, such as the url of a webpage or a link to an image. Examples would be:
a href= www."udacity.com" Surround that with angled brackets, follow it with some text content and then an end tag and you will have a link. anchors are the a's inside of angled brackets: they are for making links.
The image or img tag is a void tag: img src="url" alt= "txt" The alt= "some text here" gives the computers instructions on what to display to the user if the picture at the link is unavailable.
The letters src stand for source; an image or img always has a source or src just like an achor or a always has a hypertext reference or href.

Inline Elements vs. Block Elements

Inline elements occupy the same line with text when you are writing your code or HTML code. An example is the <em> element used to italicize words within a larger body of text. In contrast, block elements are written on their own lines within the code and serve to set apart whatever they enclose.

Container Elements

Container elemenents contain stuff inside of them. Examples of container elements are <div> and <span>. You can use <div> tags to nest boxs of text inside of other boxes by nesting the div tags. You can do the same with <span> tags.(span tags are inline elements; they demarcate boxes on one line). You do not see these boxes in the browser - they are invisible unless otherwise styled into view - but their effect is visible in how other elements on the webpage will appear to be demarcated and separated.

White Space

White space or empty space in the code gets ignored by the browser when displaying or rendering the webpage. Text is rendered continuously on one line unless you tell the computer otherwise with appropriate tags.

Notes on Headings and Format

Documents will often start and end with a tag that declares that it is an HTML document: <html>. The main body of a web page contents (including all tags within the body of the webpage) will be nested with a pair of tags <body> and its closing tag<body>. A section above the body is demarcated as the head with a pair of tags: <head> and <head>. Within the head tags, the title tags and title of the webpage will be specified (which the browser will render onto its tab associated with the webpage) as well as a link tag that serves to link the html code with the associated style file.

The Structure of a Webpage

HTML, CSS, and Javascript

Where HTML is analogous to the floor plan of a house, the CSS file linked to the webpage if analougous to the stye plans and decoration of a house, and javascript is analogous to the machines, appliances, and gadgets within the house.

Document Object Model

Webbrowsers use HTML tags to built a tree-like structure of the nested hierarchies and sections that make up the webpage. This tree-like structure is known as theDocument Object Model or DOM. When you use a browser to view the HTML code that is 'under the hood' of a webpage, the browser will tag the various separate HTML elements with little triangles, which, when selected or clicked on, with expand to show the element in further detail, sort of like dissecting the webpage.

Boxes

Stated most simply, HTML structures a webpage into a bunch of boxes, some of which are separate and some of which can contain...other boxes. When you see shapes on a webpage that are not boxes(e.g. circles, elipses, triangles) this has been accomplished by using the CSS file linked to the HTML document for the page.

Efficient Styling with CSS

Cascading Style Sheets Prevent Repetition

By using the code for Cascading Style Sheets (CSS) and linking CSS files with their HTML documents, web developers accomplish a number of efficiences for themselves, the internet, and the browser that is is ultimately rendering the page for the viewer. CSS code allows the developer to apply the same styling to multiple HTML elements. For example, the developer could apply the same styling to all <h1> tags or to all paragraphes or to all <div> items that share the attribute/value combination class="Desciption" or some other similar attribute. By avoiding repetition in coding for style, CSS helps to reduce the number of 'moving parts' in the code of a a webpage, the coder has less of a change to introduce errors (because) they are not typing as much, they will detect errors sooner (because a shorter body of code is more easy to read and review).

Abstraction Ennacted

The benefits of CSS illustrate the centrality of abstraction or generalizability in computer science and programming. Coders, the code they write, and the computers that run their code can all accomplish more with less to the degree that like elements are grouped together and ennacted upon in concert via abstraction.

Key Techniques and Methods

CSS Techniques

The specific a given CSS selector is, the more it will override other styles and selectors. For example, the CSS ID selector is very specific: in HTML it will look like <div id="some_value">, and the corresponding CSS selector would look like this: #some_value. That is, a hash symbol followed by the value of the id attribute followed by the curly braces. The class selector is less specific, but allows for a certain amount of specificity. In HTML the attribute in the tag looks like this: class="some_value". The corresponding CSS selector uses a kind of dot-notation and would look like this: .some_value, followed by the opening curley brace.
